# 요구사항 문서

## 소개

사용자 맞춤형 대시보드 기능은 현재 하드코딩된 AI 학생 비서 앱의 모든 데이터를 사용자가 직접 관리하고 커스터마이징할 수 있도록 전환하는 기능이다. 프로필 정보, 일정, 체크리스트, 학업 데이터, 행정 정보 등을 사용자가 자유롭게 추가, 수정, 삭제할 수 있으며, 개선된 프론트엔드 UI를 통해 더 나은 사용자 경험을 제공한다. 모든 데이터는 브라우저 localStorage에 영속적으로 저장된다.

## 용어 정의

- **앱**: AI 학생 비서 React 애플리케이션 전체
- **데이터_저장소**: 브라우저 localStorage를 활용한 클라이언트 측 영속 저장 계층
- **프로필_관리자**: 사용자 프로필 정보(이름, 대학교, 학과, 학년, 학점 등)를 편집하는 UI 컴포넌트
- **일정_관리자**: 일정 항목을 추가, 수정, 삭제하는 UI 컴포넌트
- **체크리스트_관리자**: 체크리스트 항목을 추가, 수정, 삭제, 토글하는 UI 컴포넌트
- **학업_관리자**: 과목별 학점, 시험 일정, 과제 진행률을 관리하는 UI 컴포넌트
- **행정_관리자**: 교내외 활동, 공모전, 커뮤니케이션 기록을 관리하는 UI 컴포넌트
- **브리핑_관리자**: 랜딩 페이지의 브리핑 메시지를 관리하는 UI 컴포넌트
- **설정_페이지**: 사용자 설정과 데이터 관리 기능을 모아놓은 전용 페이지
- **모달**: 데이터 입력 및 수정을 위한 오버레이 폼 UI
- **빈_상태_화면**: 데이터가 없을 때 표시되는 안내 UI

## 요구사항

### 요구사항 1: 클라이언트 측 데이터 영속 저장

**사용자 스토리:** 사용자로서, 내가 입력한 데이터가 브라우저를 닫았다가 다시 열어도 유지되기를 원한다. 그래야 매번 데이터를 다시 입력하지 않아도 된다.

#### 인수 조건

1. THE 데이터_저장소 SHALL localStorage를 사용하여 모든 사용자 데이터를 JSON 형식으로 저장한다
2. WHEN 앱이 처음 로드될 때, THE 데이터_저장소 SHALL localStorage에서 저장된 데이터를 읽어 앱 상태를 복원한다
3. WHEN 사용자가 데이터를 추가, 수정, 또는 삭제할 때, THE 데이터_저장소 SHALL 변경된 데이터를 즉시 localStorage에 반영한다
4. IF localStorage에 저장된 데이터가 없거나 파싱에 실패하면, THEN THE 데이터_저장소 SHALL 빈 기본값으로 앱을 초기화한다
5. THE 데이터_저장소 SHALL 프로필, 일정, 체크리스트, 학업, 행정, 브리핑 데이터를 각각 독립된 키로 저장한다

### 요구사항 2: 프로필 정보 관리

**사용자 스토리:** 사용자로서, 내 이름, 대학교, 학과, 학년, 학점 등의 프로필 정보를 직접 편집하고 싶다. 그래야 내 실제 정보가 대시보드에 반영된다.

#### 인수 조건

1. THE 프로필_관리자 SHALL 이름, 대학교, 학과, 학년, 학점(현재/최대), 이수학점(현재/최대) 필드를 편집 가능한 폼으로 제공한다
2. WHEN 사용자가 프로필 편집 버튼을 클릭할 때, THE 프로필_관리자 SHALL 현재 프로필 데이터가 채워진 편집 모달을 표시한다
3. WHEN 사용자가 프로필 편집을 저장할 때, THE 프로필_관리자 SHALL 변경된 정보를 랜딩 페이지와 대시보드 페이지에 즉시 반영한다
4. IF 사용자가 필수 필드(이름, 대학교)를 비워둔 채 저장을 시도하면, THEN THE 프로필_관리자 SHALL 해당 필드에 유효성 오류 메시지를 표시한다
5. WHEN 앱에 저장된 프로필 데이터가 없을 때, THE 프로필_관리자 SHALL 빈_상태_화면과 함께 프로필 설정 안내를 표시한다

### 요구사항 3: 일정 데이터 관리

**사용자 스토리:** 사용자로서, 내 일정을 직접 추가, 수정, 삭제하고 싶다. 그래야 캘린더와 다가오는 일정 목록에 내 실제 일정이 표시된다.

#### 인수 조건

1. WHEN 사용자가 일정 추가 버튼을 클릭할 때, THE 일정_관리자 SHALL 날짜, 시간, 제목, 태그(수업/과제/시험/팀플/활동) 입력 필드가 포함된 모달을 표시한다
2. WHEN 사용자가 일정을 저장할 때, THE 일정_관리자 SHALL 해당 일정을 캘린더의 이벤트 표시와 다가오는 일정 목록에 즉시 반영한다
3. WHEN 사용자가 기존 일정 항목을 클릭할 때, THE 일정_관리자 SHALL 해당 일정의 수정 모달을 표시한다
4. WHEN 사용자가 일정 삭제를 확인할 때, THE 일정_관리자 SHALL 해당 일정을 캘린더와 다가오는 일정 목록에서 즉시 제거한다
5. IF 사용자가 날짜 또는 제목 필드를 비워둔 채 저장을 시도하면, THEN THE 일정_관리자 SHALL 해당 필드에 유효성 오류 메시지를 표시한다
6. THE 일정_관리자 SHALL 다가오는 일정 목록을 날짜 오름차순으로 정렬하여 표시한다
7. WHEN 저장된 일정 데이터가 없을 때, THE 일정_관리자 SHALL 빈_상태_화면과 함께 일정 추가 안내를 표시한다

### 요구사항 4: 체크리스트 관리

**사용자 스토리:** 사용자로서, 체크리스트 항목을 자유롭게 추가, 수정, 삭제하고 싶다. 그래야 내 할 일 목록을 직접 관리할 수 있다.

#### 인수 조건

1. WHEN 사용자가 체크리스트 추가 버튼을 클릭할 때, THE 체크리스트_관리자 SHALL 새 항목 텍스트를 입력할 수 있는 인라인 입력 필드를 표시한다
2. WHEN 사용자가 체크리스트 항목을 클릭할 때, THE 체크리스트_관리자 SHALL 해당 항목의 완료 상태를 토글한다
3. WHEN 사용자가 체크리스트 항목의 삭제 버튼을 클릭할 때, THE 체크리스트_관리자 SHALL 해당 항목을 목록에서 즉시 제거한다
4. IF 사용자가 빈 텍스트로 체크리스트 항목 추가를 시도하면, THEN THE 체크리스트_관리자 SHALL 항목을 추가하지 않고 입력 필드에 포커스를 유지한다
5. THE 체크리스트_관리자 SHALL 완료되지 않은 항목을 목록 상단에, 완료된 항목을 하단에 표시한다
6. WHEN 저장된 체크리스트 데이터가 없을 때, THE 체크리스트_관리자 SHALL 빈_상태_화면과 함께 항목 추가 안내를 표시한다

### 요구사항 5: 학업 데이터 관리

**사용자 스토리:** 사용자로서, 과목별 학점, 시험 일정, 과제 진행률을 직접 관리하고 싶다. 그래야 내 실제 학업 현황이 대시보드에 반영된다.

#### 인수 조건

1. WHEN 사용자가 과목 추가 버튼을 클릭할 때, THE 학업_관리자 SHALL 과목명, 학점(숫자), 시험일, 시험 범위 입력 필드가 포함된 모달을 표시한다
2. WHEN 사용자가 과제 추가 버튼을 클릭할 때, THE 학업_관리자 SHALL 과제명, 관련 과목, 마감일, 진행률(0-100) 입력 필드가 포함된 모달을 표시한다
3. WHEN 사용자가 과목 또는 과제를 저장할 때, THE 학업_관리자 SHALL 학업 성취도, 시험 준비, 진행중 과제 섹션에 즉시 반영한다
4. WHEN 사용자가 기존 과목 또는 과제를 클릭할 때, THE 학업_관리자 SHALL 해당 항목의 수정 모달을 표시한다
5. WHEN 사용자가 과목 또는 과제 삭제를 확인할 때, THE 학업_관리자 SHALL 해당 항목을 학업 페이지에서 즉시 제거한다
6. THE 학업_관리자 SHALL 전체 과목의 평균 학점을 자동으로 계산하여 표시한다
7. IF 사용자가 과목명 필드를 비워둔 채 저장을 시도하면, THEN THE 학업_관리자 SHALL 해당 필드에 유효성 오류 메시지를 표시한다
8. WHEN 저장된 학업 데이터가 없을 때, THE 학업_관리자 SHALL 빈_상태_화면과 함께 과목 추가 안내를 표시한다

### 요구사항 6: 행정 정보 관리

**사용자 스토리:** 사용자로서, 교내외 활동, 공모전, 커뮤니케이션 기록을 직접 관리하고 싶다. 그래야 내 실제 활동 내역이 반영된다.

#### 인수 조건

1. WHEN 사용자가 활동 추가 버튼을 클릭할 때, THE 행정_관리자 SHALL 활동명, 설명, 태그(동아리/공모전/봉사/학회), 아이콘 선택 필드가 포함된 모달을 표시한다
2. WHEN 사용자가 공모전 추가 버튼을 클릭할 때, THE 행정_관리자 SHALL 공모전명, 일정, 상금/혜택, 태그 입력 필드가 포함된 모달을 표시한다
3. WHEN 사용자가 활동 또는 공모전을 저장할 때, THE 행정_관리자 SHALL 해당 항목을 행정 페이지에 즉시 반영한다
4. WHEN 사용자가 기존 활동 또는 공모전을 클릭할 때, THE 행정_관리자 SHALL 해당 항목의 수정 모달을 표시한다
5. WHEN 사용자가 활동 또는 공모전 삭제를 확인할 때, THE 행정_관리자 SHALL 해당 항목을 행정 페이지에서 즉시 제거한다
6. WHEN 저장된 행정 데이터가 없을 때, THE 행정_관리자 SHALL 빈_상태_화면과 함께 활동 추가 안내를 표시한다

### 요구사항 7: 브리핑 메시지 관리

**사용자 스토리:** 사용자로서, 랜딩 페이지의 브리핑 메시지를 직접 추가, 수정, 삭제하고 싶다. 그래야 내가 원하는 알림과 메모를 브리핑 슬라이더에 표시할 수 있다.

#### 인수 조건

1. WHEN 사용자가 브리핑 추가 버튼을 클릭할 때, THE 브리핑_관리자 SHALL 시간 라벨과 메시지 내용 입력 필드가 포함된 모달을 표시한다
2. WHEN 사용자가 브리핑을 저장할 때, THE 브리핑_관리자 SHALL 해당 브리핑을 랜딩 페이지 슬라이더에 즉시 반영한다
3. WHEN 사용자가 기존 브리핑 항목을 클릭할 때, THE 브리핑_관리자 SHALL 해당 브리핑의 수정 모달을 표시한다
4. WHEN 사용자가 브리핑 삭제를 확인할 때, THE 브리핑_관리자 SHALL 해당 브리핑을 슬라이더에서 즉시 제거한다
5. WHEN 저장된 브리핑 데이터가 없을 때, THE 브리핑_관리자 SHALL 슬라이더 영역에 브리핑 추가 안내를 표시한다

### 요구사항 8: 대시보드 요약 카드 동적 반영

**사용자 스토리:** 사용자로서, 대시보드의 요약 카드(일정/학업/행정)가 내가 입력한 실제 데이터를 기반으로 표시되기를 원한다. 그래야 대시보드에서 내 현황을 한눈에 파악할 수 있다.

#### 인수 조건

1. THE 앱 SHALL 대시보드의 일정 요약 카드에 사용자가 등록한 일정 중 가장 가까운 3건을 날짜순으로 표시한다
2. THE 앱 SHALL 대시보드의 학업 요약 카드에 사용자가 등록한 진행중 과제와 다가오는 시험 중 최대 3건을 표시한다
3. THE 앱 SHALL 대시보드의 행정 요약 카드에 사용자가 등록한 최근 활동과 공모전 중 최대 3건을 표시한다
4. THE 앱 SHALL 대시보드의 프로필 헤더에 사용자가 입력한 프로필 정보를 표시한다
5. THE 앱 SHALL 대시보드의 이번 주 요약 필(weekly pill)에 해당 주의 과제 수와 시험 수를 사용자 데이터 기반으로 계산하여 표시한다
6. WHEN 특정 영역에 사용자 데이터가 없을 때, THE 앱 SHALL 해당 요약 카드에 데이터 추가를 유도하는 안내 메시지를 표시한다

### 요구사항 9: 설정 및 데이터 초기화

**사용자 스토리:** 사용자로서, 내 데이터를 한꺼번에 초기화하거나 내보낼 수 있는 설정 페이지가 있으면 좋겠다. 그래야 데이터를 쉽게 관리할 수 있다.

#### 인수 조건

1. THE 앱 SHALL 설정_페이지로 이동할 수 있는 네비게이션 요소를 제공한다
2. WHEN 사용자가 전체 데이터 초기화 버튼을 클릭할 때, THE 설정_페이지 SHALL 확인 대화상자를 표시한다
3. WHEN 사용자가 데이터 초기화를 확인할 때, THE 설정_페이지 SHALL localStorage의 모든 앱 데이터를 삭제하고 앱을 초기 상태로 되돌린다
4. THE 설정_페이지 SHALL 각 데이터 영역(프로필, 일정, 체크리스트, 학업, 행정, 브리핑)별 개별 초기화 옵션을 제공한다

### 요구사항 10: 프론트엔드 UI 개선

**사용자 스토리:** 사용자로서, 더 직관적이고 사용하기 편한 UI를 원한다. 그래야 데이터 관리 작업을 쉽고 빠르게 수행할 수 있다.

#### 인수 조건

1. THE 앱 SHALL 모든 데이터 관리 모달에 일관된 디자인(border-radius, shadow, 색상)을 적용한다
2. THE 앱 SHALL 데이터 추가 버튼을 각 섹션에 시각적으로 명확하게 배치한다
3. THE 앱 SHALL 삭제 동작에 확인 대화상자를 표시하여 실수로 인한 데이터 손실을 방지한다
4. THE 앱 SHALL 모든 모달과 폼 요소에 860px 이하 화면에서 적절한 반응형 레이아웃을 적용한다
